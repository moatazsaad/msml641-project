# User Testing Notes

## Goal

Test if students understand the TerpLoad risk report

## What we tested

A simple command-line prototype where a student enters planned courses and gets:

- risk level
- risk score
- reasons

## Example

```text
CMSC330, CMSC351, STAT400
```

## Questions to ask students

- Is the risk level clear?
- Are the reasons useful?
- Would this help before registration?
- What is confusing or missing?

## Limitations

- This is only a prototype
- It uses sample workload signals
- It is not the final trained model
